# ZANDI - 잔디 재배 사업용 간편 장부 시스템 요구사항 정의서 (PRD)

- **버전**: v1.2.0
- **최종 업데이트 날짜**: 2026-06-16
- **문서 상태**: 최신 구현 현황 반영 완료

---

본 문서는 10년 이상의 회계사 경험을 바탕으로, 잔디 재배 및 판매 사업의 특성(거래처별 단가 상이, 인부 일당/반일당 관리, 철저한 1인 비공개 보안, 임대 농지 도지세 관리)을 반영하여 설계한 간편 장부 시스템의 요구사항 정의서입니다.

---

## 1. 프로젝트 개요
- **프로젝트 명**: ZANDI Ledger (잔디 재배 사업 장부)
- **대상 사용자**: 잔디 재배 사업주 1인 (1인 단독 사용 시스템, 타인 접근 불가)
- **주요 목적**: 
  - 거래처별 상이한 잔디 판매 금액 및 미수금 관리
  - 거래처 목록 수동 드래그 앤 드롭 정렬 기능 및 입력 편의성 증대
  - 인부들의 하루/반나절 출근 기록 및 날짜 필터 기반 인건비 일괄 정산 관리
  - 농지 임대료(도지세)의 동네별 분류 및 연도별 지급 현황 관리
  - 회계 지식이 없어도 한눈에 매출, 지출, 순이익을 파악할 수 있는 직관적인 대시보드 제공

---

## 2. 기술 스택
- **Core (Frontend)**: HTML5, Pure JavaScript (Vanilla JS), CSS3 (디자인 최적화)
- **Backend/DB**: Supabase (Database - Realtime Sync support, Auth)
- **Deployment**: Vercel
- **보안/인증**: 단일 마스터 비밀번호 인증 기반 프론트엔드 락(Lock) 구현 및 Supabase Row Level Security(RLS)를 통한 데이터 격리

---

## 3. 회계/경영적 관점의 핵심 설계 및 기능 요구사항

### 3.1. 보안 및 1인 제한 (Security & Privacy)
- **1인 전용 시스템 Lock**: 마스터 비밀번호 기반의 로그인 오버레이 화면을 통해 외부인 접근을 원천 차단.
- **데이터 격리**: 다른 사용자가 절대 데이터에 접근할 수 없도록 DB RLS(Row Level Security) 설정 및 세션 잠금 유지.

### 3.2. 거래처 관리 및 정렬 (Customer & Price Management)
- **거래처 등록/수정/삭제**: 거래처명, 연락처, 메모 입력 기능.
- **거래처별 잔디 단가 설정**: 
  - 거래처마다 잔디 거래 단가(예: 1818, 1818t, 3030, 3030t, 4060, 평당 단가, 여유분 단가 등)가 다르므로, **거래처 등록 시 규격별 판매 단가를 개별 설정**할 수 있게 함.
  - 판매 등록 시 거래처를 선택하면 설정된 단가가 자동으로 입력되되, 상황에 따라 즉시 수정 가능하도록 설계.
- **거래처 목록 수동 정렬 (Manual Sorting & Drag-and-Drop)**:
  - 자주 거래하는 거래처를 상단에 배치하기 위해 위/아래 이동(Chevron Up/Down) 버튼뿐만 아니라 **마우스 및 모바일 터치 드래그 앤 드롭**을 지원하여 수동으로 순서를 정렬하고 실시간 반영.
  - 시스템이 임의로 순서를 재조정(자동 정렬)하지 않으며, 사용자가 정렬해 둔 커스텀 순서(`sort_order` 컬럼 기반)가 데이터베이스에 영구 고정되어 유지됨.

### 3.3. 잔디 판매 대장 및 수금 관리 (Sales Ledger)
- **간편 등록 폼**: 판매 일자, 거래처 선택, 판매 수량(자동 단가 연동), 총 판매금액(수량 × 단가 자동 계산), 수금 상태, 메모.
- **판매 입력 시 거래처 선택값 유지**:
  - 판매 내역을 등록하여 입력 폼이 리셋되어도 **직전에 입력한 거래처 선택값이 그대로 유지**되도록 편의 기능 제공 (추가 선택 없이 수량과 단가만 연속 입력 가능).
- **분할 및 이월 미수금 관리 (Payments History)**:
  - 한 매출 건에 대해 여러 차례 나누어 수금할 수 있도록 결제 이력(Payments) 배열을 통한 상세 수금 내역 관리.
  - 이월된 미수금에 대해 일괄 및 분할 입금 처리 기능 제공.
  - 거래처별 실시간 미수금 잔액 합산 요약 정보 제공.

### 3.4. 인부 출근 및 인건비 대장 (Labor & Attendance Ledger)
- **인부 등록/관리**: 인부 이름, 기본 일당(기본 단가) 설정.
- **출근 기록 등록**: 
  - 근무 일자, 인부 선택.
  - **근무 형태 선택**: `하루 근무(1.0)` / `반나절 근무(0.5)` 선택 기능.
  - **인건비 자동 계산**: 기본 일당 × 근무 형태(1.0 또는 0.5)로 지급할 금액 자동 계산.
  - **지급 상태**: 지급 완료 / 미지급 처리.
- **필터링 기반 인건비 일괄 지급 (Bulk Wage Payment)**:
  - 인부별, 날짜 범위(예: 10일 단위 정산) 등의 필터 조건을 지정한 후 **"선택한 내역 일괄 지급 완료"** 버튼을 통해 여러 건의 인건비 미지급 내역을 한 번에 정산(지급 완료) 처리하는 기능 제공.

### 3.5. 임대 대장 및 도지세 관리 (Rents & Lease Management)
- **임대 농지 정보 관리**: 임대인(소유주)명, 연락처, 주소(지번), 면적(평수), 임대 금액(도지세), 지급 은행 계좌, 메모 등록 및 수정.
- **동네(지역) 분류**: 
  - 임대 농지를 동네 단위(최대 4개 동네)로 분류하고 필터링.
  - 각 동네의 명칭은 로컬 브라우저 저장소(LocalStorage)를 통해 사용자가 직접 수정하여 사용하며, 기기 단위로 설정 상태가 안전하게 유지됨.
- **연도별 도지세 지급 현황**: 연도별 지급 상태(지급 완료 / 미지급) 관리 및 지급 일자 기록.

### 3.6. 회계 대시보드 (Dashboard & Reports)
- **종합 손익 요약 (Net Income)**: 
  - 총 매출 (수금 완료 + 미수금)
  - 총 인건비 지출 (지급 완료 + 미지급)
  - 예상 순이익 (총 매출 - 총 인건비)
- **일정별 잔디 출하 현황 (캘린더)**: 월별 달력을 통해 날짜별 잔디 납품 건수와 거래처별 납품 요약을 시각적으로 표시.
- **미수금 신속 조회**: 미수금이 남아있는 거래처 목록과 연락처, 미수금 액수를 메인 대시보드에 실시간으로 노출하여 수금 독려 편의성 제공.

## 4. UI/UX 디자인 요구사항
- **Aesthetics**: 프리미엄 다크 모드/그린 테마 (잔디밭을 상징하는 깊이 있는 HSL 그린 톤 포인트 컬러 적용).
- **입력 편의성**: 스마트폰으로 밭에서 바로 입력할 수 있도록 모바일에 최적화된 터치 친화적 반응형 UI/UX.
- **시각화**: 달력 뷰, 진행 표시 바, 경고 아이콘 등을 활용하여 자금 흐름과 미수 상태를 직관적으로 전달.
- **실시간 동기화 상태 표시**: Supabase 연동 상태(실시간 연동중 / 연동 오류 / 로컬 전용 모드)를 상단 인디케이터로 실시간 표시.
