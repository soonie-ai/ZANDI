# ZANDI - 잔디 재배 사업용 간편 장부 시스템 요구사항 정의서 (PRD)

- **버전**: v2.4.0
- **최종 업데이트 날짜**: 2026-06-24
- **문서 상태**: 일별 상세 통합 모달 개편 및 최근 변경 알림 피드 시스템 추가 완료 (v2.4.0)

---

본 문서는 잔디 재배 및 판매 사업의 특성(거래처별 단가 상이, 인부 일당/반일당 관리, 사장님 부부 2인 전용 접근 보안, 임대 농지 도지세 관리)을 반영하여 설계한 간편 장부 시스템의 요구사항 정의서입니다.

---

## 1. 프로젝트 개요
- **프로젝트 명**: ZANDI Ledger (잔디 재배 사업 장부)
- **대상 사용자**: 잔디 재배 사업주 부부 2인 (`moonie`, `soonie`)
- **주요 목적**: 
  - 거래처별 상이한 잔디 판매 금액 및 미수금 관리
  - 거래처 목록 수동 드래그 앤 드롭 정렬 기능 및 입력 편의성 증대
  - 인부들의 하루/반나절 출근 기록 및 날짜 필터 기반 인건비 일괄 정산 관리
  - 농지 임대료(도지세)의 동네별 분류 및 연도별 지급 현황 관리
  - 부부 전용 2인 보안 로그인 환경을 통한 실시간 다중 기기 동기화 및 외부 접근 차단
  - 매출, 지출, 순이익을 한눈에 파악할 수 있는 직관적인 대시보드 제공

---

## 2. 기술 스택
- **Core (Frontend)**: HTML5, Pure JavaScript (Vanilla JS), CSS3 (디자인 최적화)
- **Backend/DB/Auth**: Supabase (Database & Auth - Realtime Sync support)
- **Deployment**: Vercel (CDN Cache Control 최적화 적용)
- **보안/인증**: Supabase Auth (이메일/비밀번호 방식 기반, 내부 가상 도메인 자동 처리)
- **구조 최적화**: 5개의 역할 분할 모듈(`state.js`, `utils.js`, `db.js`, `ui.js`, `app.js`) 구조

## 3. 핵심 설계 및 기능 요구사항

### 3.1. 보안 및 로그인 (Security & Authentication)
- **사장님 부부 2인 전용 로그인**:
  - 시스템 접근을 위해 아이디(`moonie`, `soonie`)와 6자리 이상의 숫자로 이루어진 비밀번호로 로그인하는 전용 오버레이 제공.
  - 로그인 시 내부적으로 가상 도메인(`@zandi.com`)을 결합하여 Supabase Auth API를 통해 본인 인증을 수행하며, 로그아웃 시 인증 세션을 파기함.
  - 보안을 위해 회원가입 기능은 배제하고 지정된 계정만 로그인이 가능하도록 차단.
- **Supabase 연결 정보 고정**:
  - API 연결 키 유실로 인한 연동 해제 및 데이터 유실을 방지하기 위해 Supabase URL 및 anon key를 소스 코드 내에 고정하여 초기 설정 폼 없이 안정적인 접속 보장.

### 3.2. 거래처 관리 및 정렬 (Customer & Price Management)
- **거래처 등록/수정/삭제**: 거래처명, 연락처, 메모 입력 기능.
- **거래처별 잔디 단가 설정**: 
  - 거래처마다 잔디 거래 단가(예: 1818, 1818t, 3030, 3030t, 4060, 평당 단가, 여유분 단가 등)가 다르므로, 거래처 등록 시 규격별 판매 단가를 개별 설정.
  - 판매 등록 시 거래처를 선택하면 설정된 단가가 자동으로 입력되되, 상황에 따라 즉시 수정 가능하도록 설계.
- **거래처 목록 수동 정렬 (Manual Sorting & Drag-and-Drop)**:
  - 자주 거래하는 거래처를 상단에 배치하기 위해 위/아래 이동 버튼 및 마우스/모바일 터치 드래그 앤 드롭 정렬 지원.
  - 사용자가 정렬해 둔 커스텀 순서(`sort_order` 컬럼 기반)가 데이터베이스에 영구 고정되어 유지됨.

### 3.3. 잔디 판매 대장 및 수금 관리 (Sales Ledger)
- **간편 등록 폼**: 판매 일자, 거래처 선택, 판매 수량(자동 단가 연동), 총 판매금액(수량 × 단가 자동 계산), 수금 상태, 메모.
- **판매 입력 시 거래처 선택값 유지**:
  - 판매 내역을 등록하여 입력 폼이 리셋되어도 직전에 입력한 거래처 선택값이 그대로 유지되도록 편의 제공.
- **분할 및 이월 미수금 관리 (Payments History)**:
  - 한 매출 건에 대해 여러 차례 나누어 수금할 수 있도록 결제 이력(Payments) 배열을 통한 상세 수금 내역 관리.
  - 이월된 미수금에 대해 일괄 및 분할 입금 처리 기능 제공.
  - 거래처별 실시간 미수금 잔액 합산 요약 정보 제공.
- **수금/미수금 상태 가독성 대폭 향상 (v2.1.0)**:
  - 수금 상태(수금완료/부분수금/미수금) 배지 크기를 키우고, 미수 상태 시 미세하게 깜빡이는 애니메이션(`animate-pulse`)을 주어 시각적으로 강조.
  - 불필요하고 자잘한 텍스트를 정리하고, 수금 완료 시에는 완료 금액을 표시하며 미수 상황일 때는 **로즈 핑크 계열의 굵고 큰 폰트로 미납 금액 단독 노출** (`1,500,000원 미납` 형태).
- **판매 메모 전용 모달창 적용 (v2.1.0)**:
  - 기존 브라우저 기본 경고창(`alert`) 대신, 어플리케이션 테마와 어우러지는 다크 글래스모피즘 스타일의 **커스텀 메모 모달창**(`#sale-note-modal`)을 통해 메모를 편안하게 조회할 수 있도록 개선.

### 3.4. 인부 출근 및 인건비 대장 (Labor & Attendance Ledger)
- **인부 등록/관리 및 개별 단가 설정**: 
  - 인부 이름, 하루 일당(1.0 기준), 반일당(반나절 0.5 기준) 개별 설정 기능 지원.
  - 등록된 인부의 이름, 일당, 반일당 정보를 언제든 수정할 수 있는 "인부 정보 수정 모달" 지원.
- **출근 기록 등록**: 
  - 근무 일자, 인부 선택.
  - **근무 형태 선택**: `하루 근무(1.0)` / `반나절 근무(0.5)`.
  - **인건비 자동 계산 및 유연성**: 
    - 하루 선택 시 설정된 하루 일당 적용, 반나절 선택 시 사전에 설정된 반일당이 기본값으로 자동 입력되도록 구현.
    - 출근 등록 현장에서 특정 날짜에 따라 일당 금액을 수동으로 편집하여 즉시 기록할 수 있는 유연한 입력 폼 제공.
  - **지급 상태**: 지급 완료 / 미지급 처리.
- **필터링 기반 인건비 일괄 지급 (Bulk Wage Payment)**:
  - 인부별, 날짜 범위 등의 필터 조건을 지정한 후 "선택한 내역 일괄 지급 완료" 버튼을 통해 여러 건의 인건비 미지급 내역을 한 번에 정산(지급 완료) 처리하는 기능 제공.
- **월별 인건비 지급 내역 집계 (Monthly Wage Report)**:
  - 인부 명단 헤더 영역에 "월별 급여 집계 보기" 팝업 모달 버튼을 신설하여, 선택한 월(YYYY-MM)에 출근한 인부들의 총 근무일수, 총 인건비, 실제 지급완료 금액, 미지급 금액을 개별 집계 및 전체 총합 행으로 팝업창에서 한눈에 조회할 수 있는 기능 제공.
- **인부출근대장 달력 일별 상세 조회 모달창 추가 (v2.2.0)**:
  - 인부출근대장의 달력 뷰에서 지정된 날짜를 클릭하면 해당 일자에 출근한 인부들의 현황(이름, 근무 형태, 기준 일당, 정산 금액, 지급 여부)을 테이블 형태로 상세히 확인할 수 있는 커스텀 상세 팝업 모달창(`#attendance-day-details-modal`) 기능 제공.

### 3.5. 임대 대장 및 도지세 관리 (Rents & Lease Management)
- **임대 농지 정보 관리**: 임대인(소유주)명, 연락처, 주소(지번), 면적(평수), 임대 금액(도지세), 지급 은행 계좌, 메모 등록 및 수정.
- **동네(지역) 분류**: 
  - 임대 농지를 동네 단위(최대 4개 동네)로 분류하고 필터링.
  - 각 동네의 명칭은 로컬 브라우저 저장소(LocalStorage)를 통해 사용자가 직접 수정하여 사용하며, 기기 단위로 설정 상태가 안전하게 유지됨.
- **연도별 도지세 지급 현황**: 연도별 지급 상태(지급 완료 / 미지급) 관리 및 지급 일자 기록.

### 3.6. 농자재 지출 관리 (Agricultural Expenses Management)
- **독립 탭 카테고리 구성**: 사이드바 메뉴 대시보드 아래에 "농자재 지출 관리" 독립 탭 메뉴 신설.
- **수기 지출 등록 및 관리**: 비료, 농약 등 농가 운영 지출의 날짜, 사용처(품명), 지출 금액, 비고를 수기로 추가, 수정, 삭제(CRUD)할 수 있는 관리 기능 제공.
- **지출 집계 대시보드**: 등록된 총 지출 건수와 지출 금액의 합계를 보여주는 직관적인 현황판 카드 제공.

### 3.7. 회계 대시보드 (Dashboard & Reports)
- **종합 손익 요약 (Net Income)**: 
  - 총 매출 (수금 완료 + 미수금)
  - 총 인건비 지출 (지급 완료 + 미지급)
  - 예상 순이익 (총 매출 - 총 인건비)
- **일정별 잔디 출하 현황 (캘린더)**: 월별 달력을 통해 날짜별 잔디 납품 건수와 거래처별 납품 요약을 시각적으로 표시.
- **미수금 신속 및 정확 조회 (v2.2.0)**: 
  - 미수금이 남아있는 거래처 목록과 연락처, 미수금 액수를 메인 대시보드에 실시간으로 노출하되, **각 거래처의 '이월 미수금 잔액'(`initialDebt` - `initialDebtCollected`)을 합산하여 표시**함으로써 실제 받아야 할 돈을 누락 없이 정확하게 파악할 수 있도록 개선.
  - 대시보드 상단의 "미수금액" 요약 카드 역시 거래처별 전체 미수금(거래액 미수 + 이월 미수 잔액)의 총합이 적용되도록 계산 로직 고도화.

### 3.8. 거래명세서 발행 및 직접 편집 (v2.3.0)
- **거래처별 거래명세서 팝업 (모달) 탑재**:
  - 판매 대장 탭 내에서 거래처와 기간을 조건으로 정렬하여 거래명세서 팝업창을 띄울 수 있는 전용 발행 버튼 신설.
- **선택형 및 미수금 하이브리드 발행**:
  - 장부에서 원하는 내역만 체크박스로 수동 체크하여 발행 시 해당 건만 수집하여 발행.
  - 체크박스 미체크 상태에서 일괄 발행 시, 수금이 완료된 건은 제외하고 **오직 미수금(수금 잔액)이 남아있는 건들만 자동으로 수집**하여 청구 전용 명세서 구성.
- **실제 사업자등록 정보 고정 및 자동 저장**:
  - 사업자등록증 판독 데이터(상호: 양지잔디, 대표자: 유수경, 등록번호: 601-98-76580, 주소: 경남 사천시 곤명면 서실1길 4, 계좌번호: 농협 351-1234-1479-83)를 기본 고정값으로 설정 및 불필요한 팩스 칸 제거.
  - 사용자가 명세서상에서 대표자 정보나 계좌번호를 직접 클릭해 수정 시 브라우저 `localStorage`에 자동 저장되어 영구 보존.
- **가로 너비 고정 및 품명/규격 분리**:
  - `품명 및 규격` 열을 `품명`과 `규격` 2개 열로 분리. 품명은 기본적으로 **"재배잔디"**로 자동 고정되며 규격 칸에는 규격명 기입.
  - A4 가로 인쇄 비율에 맞춰 표의 각 열 가로 너비(%) 비율을 고정하여 가독성 최적화.
- **모달 내부 실시간 규격 필터 버튼**:
  - 명세서 모달 헤더에 규격 토글 탭(전체, 1818, 3030 등) 추가. 클릭 시 즉석에서 해당 규격의 거래만 실시간 필터링되어 보이고, 동시에 규격별 총 장수 요약과 공급가액 합계가 자동 재계산됨. (합계 수량 칸 직접 수정 가능)
- **화면 캡처용 삭제 버튼 숨기기 스위치**:
  - 폰 화면으로 그대로 캡처해 카카오톡으로 보낼 때 지저분한 '삭제' 열과 'X' 버튼을 숨길 수 있는 토글 스위치 지원.
- **인쇄 전용 스타일링 (`@media print`) 적용**:
  - 인쇄/PDF 저장 호출 시, 다크 모드 배경색과 버튼 등 명세서 본체 외의 모든 화면 요소를 가리고, 깨끗한 A4 세로 규격의 백색 종이 양식으로 출력되도록 구성.

### 3.9. 일별 통합 상세 조회 및 최근 변경 알림 피드 (v2.4.0)
- **일별 통합 상세 모달 (탭 전환)**:
  - 대시보드 캘린더나 인부출근대장 캘린더 내의 날짜를 클릭하면, 판매 내역과 인부 출근 현황을 **한 공간에서 탭([판매 내역], [인부 내역])을 전환하며** 한눈에 모아볼 수 있는 일별 통합 상세 모달 제공.
- **사이드바 종(🔔) 모양 알림 버튼 및 실시간 배지**:
  - 사이드바 내 토지 임대료 관리 메뉴와 닉네임 영역 사이에 알림 확인 버튼(`notification-bell-btn`)을 배치하고, 미확인 알림 수(빨간색 배지)를 실시간으로 노출하여 최근 변경 내용을 직관적으로 인지할 수 있도록 유도.
- **최근 변경 이력 시스템**:
  - 장부에서 발생하는 모든 CRUD(추가, 수정, 삭제) 동작이 수행될 때마다 어떤 사용자가 어떤 데이터를 다루었는지(예: `moonie님이 '홍길동' 인부의 6/24 출근을 등록했습니다.`) 자동으로 요약하여 `settings` 테이블의 `recent_activities` 키에 실시간 영구 공유.
- **사용자별 개별 확인 완료 기능**:
  - 최근 알림 확인 모달에서 개별 알림 옆의 `[확인]` 또는 상단의 `[모두 확인 완료]` 버튼을 누르면 해당 알림 ID가 각 기기의 `LocalStorage`에 '확인 완료(읽음)' 상태로 저장되어 화면 및 배지 집계에서 즉시 제외되도록 함으로써, 한 번 확인한 알림은 영구적으로 숨겨지는 깔끔한 피드 상태 유지.

## 4. UI/UX 디자인 및 예외 처리 요구사항
- **Aesthetics**: 프리미엄 다크 모드/그린 테마 (잔디밭을 상징하는 깊이 있는 HSL 그린 톤 포인트 컬러 적용).
- **입력 편의성**: 스마트폰으로 밭에서 바로 입력할 수 있도록 모바일에 최적화된 터치 친화적 반응형 UI/UX.
- **시각화**: 달력 뷰, 진행 표시 바, 경고 아이콘 등을 활용하여 자금 흐름과 미수 상태를 직관적으로 전달.
- **실시간 동기화 상태 표시**: Supabase 연동 상태(로그인 완료 및 동기화 중 / 로그아웃 / 연동 오류 등)를 상단 인디케이터로 실시간 표시.
